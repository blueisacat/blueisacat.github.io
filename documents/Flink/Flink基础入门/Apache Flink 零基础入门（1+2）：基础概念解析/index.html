    дј з»ҹжү№еӨ„зҗҶж–№жі•жҳҜжҢҒз»ӯж”¶еҸ–ж•°жҚ®пјҢд»Ҙж—¶й—ҙдҪңдёәеҲ’еҲҶеӨҡдёӘжү№ж¬Ўзҡ„дҫқжҚ®пјҢеҶҚе‘ЁжңҹжҖ§ең°жү§иЎҢжү№ж¬Ўиҝҗз®—гҖӮдҪҶеҒҮи®ҫйңҖиҰҒи®Ўз®—жҜҸе°Ҹж—¶еҮәзҺ°дәӢд»¶иҪ¬жҚўзҡ„ж¬Ўж•°пјҢеҰӮжһңдәӢд»¶иҪ¬жҚўи·Ёи¶ҠдәҶжүҖе®ҡд№үзҡ„ж—¶й—ҙеҲ’еҲҶпјҢдј з»ҹжү№еӨ„зҗҶдјҡе°Ҷдёӯд»Ӣиҝҗз®—з»“жһңеёҰеҲ°дёӢдёҖдёӘжү№ж¬ЎиҝӣиЎҢи®Ўз®—пјӣйҷӨжӯӨд№ӢеӨ–пјҢеҪ“еҮәзҺ°жҺҘж”¶еҲ°зҡ„дәӢд»¶йЎәеәҸйў еҖ’жғ…еҶөдёӢпјҢдј з»ҹжү№еӨ„зҗҶд»Қдјҡе°Ҷдёӯд»ӢзҠ¶жҖҒеёҰеҲ°дёӢдёҖжү№ж¬Ўзҡ„иҝҗз®—з»“жһңдёӯпјҢиҝҷз§ҚеӨ„зҗҶж–№ејҸд№ҹдёҚе°ҪеҰӮдәәж„ҸгҖӮ

    2.2 зҗҶжғіж–№жі•¶

    з¬¬дёҖзӮ№пјҢиҰҒжңүзҗҶжғіж–№жі•пјҢиҝҷдёӘзҗҶжғіж–№жі•жҳҜеј•ж“Һеҝ…йЎ»иҰҒжңүиғҪеҠӣеҸҜд»ҘзҙҜз§ҜзҠ¶жҖҒе’Ңз»ҙжҠӨзҠ¶жҖҒпјҢзҙҜз§ҜзҠ¶жҖҒд»ЈиЎЁзқҖиҝҮеҺ»еҺҶеҸІдёӯжҺҘж”¶иҝҮзҡ„жүҖжңүдәӢд»¶пјҢдјҡеҪұе“ҚеҲ°иҫ“еҮәгҖӮ

    з¬¬дәҢзӮ№пјҢж—¶й—ҙпјҢж—¶й—ҙж„Ҹе‘ізқҖеј•ж“ҺеҜ№дәҺж•°жҚ®е®Ңж•ҙжҖ§жңүжңәеҲ¶еҸҜд»Ҙж“ҚжҺ§пјҢеҪ“жүҖжңүж•°жҚ®йғҪе®Ңе…ЁжҺҘж”¶еҲ°еҗҺпјҢиҫ“еҮәи®Ўз®—з»“жһңгҖӮ

    з¬¬дёүзӮ№пјҢзҗҶжғіж–№жі•жЁЎеһӢйңҖиҰҒе®һж—¶дә§з”ҹз»“жһңпјҢдҪҶжӣҙйҮҚиҰҒзҡ„жҳҜйҮҮз”Ёж–°зҡ„жҢҒз»ӯжҖ§ж•°жҚ®еӨ„зҗҶжЁЎеһӢжқҘеӨ„зҗҶе®һж—¶ж•°жҚ®пјҢиҝҷж ·жүҚжңҖз¬ҰеҗҲ Continuous data зҡ„зү№жҖ§гҖӮ

    2.3 жөҒејҸеӨ„зҗҶ¶

    жөҒејҸеӨ„зҗҶз®ҖеҚ•жқҘи®ІеҚіжңүдёҖдёӘж— з©·ж— е°Ҫзҡ„ж•°жҚ®жәҗеңЁжҢҒз»ӯж”¶еҸ–ж•°жҚ®пјҢд»Ҙд»Јз ҒдҪңдёәж•°жҚ®еӨ„зҗҶзҡ„еҹәзЎҖйҖ»иҫ‘пјҢж•°жҚ®жәҗзҡ„ж•°жҚ®з»ҸиҝҮд»Јз ҒеӨ„зҗҶеҗҺдә§з”ҹеҮәз»“жһңпјҢз„¶еҗҺиҫ“еҮәпјҢиҝҷе°ұжҳҜжөҒејҸеӨ„зҗҶзҡ„еҹәжң¬еҺҹзҗҶгҖӮ

    2.4 еҲҶеёғејҸжөҒејҸеӨ„зҗҶ¶

    еҒҮи®ҫ Input Streams жңүеҫҲеӨҡдёӘдҪҝз”ЁиҖ…пјҢжҜҸдёӘдҪҝз”ЁиҖ…йғҪжңүиҮӘе·ұзҡ„ ID пјҢеҰӮжһңи®Ўз®—жҜҸдёӘдҪҝз”ЁиҖ…еҮәзҺ°зҡ„ж¬Ўж•°пјҢжҲ‘д»¬йңҖиҰҒи®©еҗҢдёҖдёӘдҪҝз”ЁиҖ…зҡ„еҮәзҺ°дәӢд»¶жөҒеҲ°еҗҢдёҖиҝҗз®—д»Јз ҒпјҢиҝҷи·ҹе…¶д»–жү№ж¬ЎйңҖиҰҒеҒҡ Group by жҳҜеҗҢж ·зҡ„жҰӮеҝөпјҢжүҖд»Ҙи·ҹ Stream дёҖж ·йңҖиҰҒеҒҡеҲҶеҢәпјҢи®ҫе®ҡзӣёеә”зҡ„ Key пјҢз„¶еҗҺи®©еҗҢж ·зҡ„ Key жөҒеҲ°еҗҢдёҖдёӘ Computation instance еҒҡеҗҢж ·зҡ„иҝҗз®—гҖӮ

    2.5 жңүзҠ¶жҖҒеҲҶеёғејҸжөҒејҸеӨ„зҗҶ¶

    еҰӮеӣҫпјҢдёҠиҝ°д»Јз Ғдёӯе®ҡд№үдәҶеҸҳж•° X пјҢ X еңЁж•°жҚ®еӨ„зҗҶиҝҮзЁӢдёӯдјҡиҝӣиЎҢиҜ»е’ҢеҶҷпјҢеңЁжңҖеҗҺиҫ“еҮәз»“жһңж—¶пјҢеҸҜд»ҘдҫқжҚ®еҸҳж•° X еҶіе®ҡиҫ“еҮәзҡ„еҶ…е®№пјҢеҚізҠ¶жҖҒ X дјҡеҪұе“ҚжңҖз»Ҳзҡ„иҫ“еҮәз»“жһңгҖӮиҝҷдёӘиҝҮзЁӢдёӯпјҢз¬¬дёҖдёӘйҮҚзӮ№жҳҜе…ҲиҝӣиЎҢдәҶзҠ¶жҖҒ Co-partitioned key by пјҢеҗҢж ·зҡ„ Key йғҪдјҡжөҒеҲ° Computation instance пјҢдёҺдҪҝз”ЁиҖ…еҮәзҺ°ж¬Ўж•°зҡ„еҺҹзҗҶзӣёеҗҢпјҢж¬Ўж•°еҚіжүҖи°“зҡ„зҠ¶жҖҒпјҢиҝҷдёӘзҠ¶жҖҒдёҖе®ҡдјҡи·ҹеҗҢдёҖдёӘ Key зҡ„дәӢд»¶зҙҜз§ҜеңЁеҗҢдёҖдёӘ Computation instance гҖӮзұ»дјјдәҺж №жҚ®иҫ“е…ҘжөҒзҡ„ Key йҮҚж–°еҲҶеҢәзҡ„зҠ¶жҖҒпјҢеҪ“еҲҶеҢәиҝӣе…Ҙ Stream д№ӢеҗҺпјҢиҝҷдёӘ Stream дјҡзҙҜз§Ҝиө·жқҘзҡ„зҠ¶жҖҒд№ҹеҸҳжҲҗ Copartiton гҖӮ

    з¬¬дәҢдёӘйҮҚзӮ№жҳҜ embeded local state backend гҖӮжңүзҠ¶жҖҒеҲҶж•ЈејҸжөҒејҸеӨ„зҗҶзҡ„еј•ж“ҺпјҢзҠ¶жҖҒеҸҜиғҪдјҡзҙҜз§ҜеҲ°йқһеёёеӨ§пјҢеҪ“ Key йқһеёёеӨҡж—¶пјҢзҠ¶жҖҒеҸҜиғҪе°ұдјҡи¶…еҮәеҚ•дёҖиҠӮзӮ№зҡ„ Memory зҡ„иҙҹиҚ·йҮҸпјҢиҝҷж—¶еҖҷзҠ¶жҖҒеҝ…йЎ»жңүзҠ¶жҖҒеҗҺз«ҜеҺ»з»ҙжҠӨе®ғпјӣеңЁиҝҷдёӘзҠ¶жҖҒеҗҺз«ҜеңЁжӯЈеёёзҠ¶еҶөдёӢпјҢз”Ё In-memory з»ҙжҠӨеҚіеҸҜгҖӮ

    3 Apache Flink зҡ„дјҳеҠҝ¶

    3.1 зҠ¶жҖҒе®№й”ҷ¶

    еҪ“жҲ‘д»¬иҖғиҷ‘зҠ¶жҖҒе®№й”ҷж—¶йҡҫе…ҚдјҡжғіеҲ°зІҫзЎ®дёҖж¬Ўзҡ„зҠ¶жҖҒе®№й”ҷпјҢеә”з”ЁеңЁиҝҗз®—ж—¶зҙҜз§Ҝзҡ„зҠ¶жҖҒпјҢжҜҸз¬”иҫ“е…Ҙзҡ„дәӢд»¶еҸҚжҳ еҲ°зҠ¶жҖҒпјҢжӣҙж”№зҠ¶жҖҒйғҪжҳҜзІҫзЎ®дёҖж¬ЎпјҢеҰӮжһңдҝ®ж”№и¶…иҝҮдёҖж¬Ўзҡ„иҜқд№ҹж„Ҹе‘ізқҖж•°жҚ®еј•ж“Һдә§з”ҹзҡ„з»“жһңжҳҜдёҚеҸҜйқ зҡ„гҖӮ

    • еҰӮдҪ•зЎ®дҝқзҠ¶жҖҒжӢҘжңүзІҫзЎ®дёҖж¬ЎпјҲ Exactly-once guarantee пјүзҡ„е®№й”ҷдҝқиҜҒпјҹ

    • еҰӮдҪ•еңЁеҲҶж•ЈејҸеңәжҷҜдёӢжӣҝеӨҡдёӘжӢҘжңүжң¬ең°зҠ¶жҖҒзҡ„иҝҗз®—еӯҗдә§з”ҹдёҖдёӘе…ЁеҹҹдёҖиҮҙзҡ„еҝ«з…§пјҲ Global consistent snapshot пјүпјҹ

    • жӣҙйҮҚиҰҒзҡ„жҳҜпјҢеҰӮдҪ•еңЁдёҚдёӯж–ӯиҝҗз®—зҡ„еүҚжҸҗдёӢдә§з”ҹеҝ«з…§пјҹ

    3.1.1 з®ҖеҚ•еңәжҷҜзҡ„зІҫзЎ®дёҖж¬Ўе®№й”ҷж–№жі•¶

    иҝҳжҳҜд»ҘдҪҝз”ЁиҖ…еҮәзҺ°ж¬Ўж•°жқҘзңӢпјҢеҰӮжһңжҹҗдёӘдҪҝз”ЁиҖ…еҮәзҺ°зҡ„ж¬Ўж•°и®Ўз®—дёҚеҮҶзЎ®пјҢдёҚжҳҜзІҫзЎ®дёҖж¬ЎпјҢйӮЈд№Ҳдә§з”ҹзҡ„з»“жһңжҳҜж— жі•дҪңдёәеҸӮиҖғзҡ„гҖӮеңЁиҖғиҷ‘зІҫзЎ®зҡ„е®№й”ҷдҝқиҜҒеүҚпјҢжҲ‘д»¬е…ҲиҖғиҷ‘жңҖз®ҖеҚ•зҡ„дҪҝз”ЁеңәжҷҜпјҢеҰӮж— йҷҗжөҒзҡ„ж•°жҚ®иҝӣе…ҘпјҢеҗҺйқўеҚ•дёҖзҡ„ Process иҝӣиЎҢиҝҗз®—пјҢжҜҸеӨ„зҗҶе®ҢдёҖз¬”и®Ўз®—еҚідјҡзҙҜз§ҜдёҖж¬ЎзҠ¶жҖҒпјҢиҝҷз§Қжғ…еҶөдёӢеҰӮжһңиҰҒзЎ®дҝқ Process дә§з”ҹзІҫзЎ®дёҖж¬Ўзҡ„зҠ¶жҖҒе®№й”ҷпјҢжҜҸеӨ„зҗҶе®ҢдёҖз¬”ж•°жҚ®пјҢжӣҙж”№е®ҢзҠ¶жҖҒеҗҺиҝӣиЎҢдёҖж¬Ўеҝ«з…§пјҢеҝ«з…§еҢ…еҗ«еңЁйҳҹеҲ—дёӯе№¶дёҺзӣёеә”зҡ„зҠ¶жҖҒиҝӣиЎҢеҜ№жҜ”пјҢе®ҢжҲҗдёҖиҮҙзҡ„еҝ«з…§пјҢе°ұиғҪзЎ®дҝқзІҫзЎ®дёҖж¬ЎгҖӮ

    3.1.2 еҲҶеёғејҸзҠ¶жҖҒе®№й”ҷ¶

    Flink дҪңдёәеҲҶеёғејҸзҡ„еӨ„зҗҶеј•ж“ҺпјҢеңЁеҲҶеёғејҸзҡ„еңәжҷҜдёӢпјҢиҝӣиЎҢеӨҡдёӘжң¬ең°зҠ¶жҖҒзҡ„иҝҗз®—пјҢеҸӘдә§з”ҹдёҖдёӘе…ЁеҹҹдёҖиҮҙзҡ„еҝ«з…§пјҢеҰӮйңҖиҰҒеңЁдёҚдёӯж–ӯиҝҗз®—еҖјзҡ„еүҚжҸҗдёӢдә§з”ҹе…ЁеҹҹдёҖиҮҙзҡ„еҝ«з…§пјҢе°ұж¶үеҸҠеҲ°еҲҶж•ЈејҸзҠ¶жҖҒе®№й”ҷгҖӮ

    • Global consistent snapshot

      е…ідәҺ Global consistent snapshot пјҢеҪ“ Operator еңЁеҲҶеёғејҸзҡ„зҺҜеўғдёӯпјҢеңЁеҗ„дёӘиҠӮзӮ№еҒҡиҝҗз®—пјҢйҰ–е…Ҳдә§з”ҹ Global consistent snapshot зҡ„ж–№ејҸе°ұжҳҜеӨ„зҗҶжҜҸдёҖз¬”ж•°жҚ®зҡ„еҝ«з…§зӮ№жҳҜиҝһз»ӯзҡ„пјҢиҝҷз¬”иҝҗз®—жөҒиҝҮжүҖжңүзҡ„иҝҗз®—еҖјпјҢжӣҙж”№е®ҢжүҖжңүзҡ„иҝҗз®—еҖјеҗҺпјҢиғҪеӨҹзңӢеҲ°жҜҸдёҖдёӘиҝҗз®—еҖјзҡ„зҠ¶жҖҒдёҺиҜҘз¬”иҝҗз®—зҡ„дҪҚзҪ®пјҢеҚіеҸҜз§°дёә Consistent snapshot пјҢеҪ“з„¶пјҢ Global consistent snapshot д№ҹжҳҜз®Җжҳ“еңәжҷҜзҡ„е»¶дјёгҖӮ

    • е®№й”ҷжҒўеӨҚ

      йҰ–е…ҲдәҶи§ЈдёҖдёӢ Checkpoint пјҢдёҠйқўжҸҗеҲ°иҝһз»ӯжҖ§еҝ«з…§жҜҸдёӘ Operator иҝҗз®—еҖјжң¬ең°зҡ„зҠ¶жҖҒеҗҺз«ҜйғҪиҰҒз»ҙжҠӨзҠ¶жҖҒпјҢд№ҹе°ұжҳҜжҜҸж¬Ўе°Ҷдә§з”ҹжЈҖжҹҘзӮ№ж—¶дјҡе°Ҷе®ғд»¬дј е…Ҙе…ұдә«зҡ„ DFS дёӯгҖӮеҪ“д»»дҪ•дёҖдёӘ Process жҢӮжҺүеҗҺпјҢеҸҜд»ҘзӣҙжҺҘд»ҺдёүдёӘе®Ңж•ҙзҡ„ Checkpoint е°ҶжүҖжңүзҡ„иҝҗз®—еҖјзҡ„зҠ¶жҖҒжҒўеӨҚпјҢйҮҚж–°и®ҫе®ҡеҲ°зӣёеә”дҪҚзҪ®гҖӮ Checkpoint зҡ„еӯҳеңЁдҪҝж•ҙдёӘ Process иғҪеӨҹе®һзҺ°еҲҶж•ЈејҸзҺҜеўғдёӯзҡ„ Exactly-once гҖӮ

    3.1.3 еҲҶж•ЈејҸеҝ«з…§пјҲDistributed Snapshotsпјүж–№жі•¶

    е…ідәҺ Flink еҰӮдҪ•еңЁдёҚдёӯж–ӯиҝҗз®—зҡ„зҠ¶еҶөдёӢжҢҒз»ӯдә§з”ҹ Global consistent snapshot пјҢе…¶ж–№ејҸжҳҜеҹәдәҺз”Ё Simple lamport жј”з®—жі•жңәеҲ¶дёӢе»¶дјёзҡ„гҖӮе·ІзҹҘзҡ„дёҖдёӘзӮ№ Checkpoint barrier пјҢ Flink еңЁжҹҗдёӘ Datastream дёӯдјҡдёҖзӣҙе®үжҸ’ Checkpoint barrier пјҢ Checkpoint barrier д№ҹдјҡ N - 1 зӯүзӯүпјҢ Checkpoint barrier N д»ЈиЎЁзқҖжүҖжңүеңЁиҝҷдёӘиҢғеӣҙйҮҢйқўзҡ„ж•°жҚ®йғҪжҳҜ Checkpoint barrier N гҖӮ

    дёҫдҫӢпјҡеҒҮи®ҫзҺ°еңЁйңҖиҰҒдә§з”ҹ Checkpoint barrier N пјҢдҪҶе®һйҷ…дёҠеңЁ Flink дёӯжҳҜз”ұ Job manager и§ҰеҸ‘ Checkpoint пјҢ Checkpoint иў«и§ҰеҸ‘еҗҺејҖе§Ӣд»Һж•°жҚ®жәҗдә§з”ҹ Checkpoint barrier гҖӮеҪ“ Job ејҖе§ӢеҒҡ Checkpoint barrier N зҡ„ж—¶еҖҷпјҢеҸҜд»ҘзҗҶи§Јдёә Checkpoint barrier N йңҖиҰҒйҖҗжӯҘеЎ«е……е·ҰдёӢи§’зҡ„иЎЁж јгҖӮ

    еҰӮеӣҫпјҢеҪ“йғЁеҲҶдәӢд»¶ж ҮдёәзәўиүІпјҢ Checkpoint barrier N д№ҹжҳҜзәўиүІж—¶пјҢд»ЈиЎЁзқҖиҝҷдәӣж•°жҚ®жҲ–дәӢд»¶йғҪз”ұ Checkpoint barrier N иҙҹиҙЈгҖӮ Checkpoint barrier N еҗҺйқўзҷҪиүІйғЁеҲҶзҡ„ж•°жҚ®жҲ–дәӢд»¶еҲҷдёҚеұһдәҺ Checkpoint barrier N гҖӮ

    еҪ“еҝ«з…§е®ҢжҲҗеҗҺз»§з»ӯеҫҖдёӢжёёиө°пјҢ Operator 2 д№ҹдјҡжҺҘж”¶еҲ°жүҖжңүж•°жҚ®пјҢз„¶еҗҺжҗңзҙў Checkpoint barrier N зҡ„ж•°жҚ®е№¶зӣҙжҺҘеҸҚжҳ еҲ°зҠ¶жҖҒпјҢеҪ“зҠ¶жҖҒж”¶еҲ° Checkpoint barrier N д№ӢеҗҺд№ҹдјҡзӣҙжҺҘеҶҷе…ҘеҲ° Checkpoint N дёӯгҖӮд»ҘдёҠиҝҮзЁӢеҲ°жӯӨеҸҜд»ҘзңӢеҲ° Checkpoint barrier N е·Із»Ҹе®ҢжҲҗдәҶдёҖдёӘе®Ңж•ҙзҡ„иЎЁж јпјҢиҝҷдёӘиЎЁж јеҸ«еҒҡ Distributed Snapshots пјҢеҚіеҲҶеёғејҸеҝ«з…§гҖӮеҲҶеёғејҸеҝ«з…§еҸҜд»Ҙз”ЁжқҘеҒҡзҠ¶жҖҒе®№й”ҷпјҢд»»дҪ•дёҖдёӘиҠӮзӮ№жҢӮжҺүзҡ„ж—¶еҖҷеҸҜд»ҘеңЁд№ӢеүҚзҡ„ Checkpoint дёӯе°Ҷе…¶жҒўеӨҚгҖӮз»§з»ӯд»ҘдёҠ Process пјҢеҪ“еӨҡдёӘ Checkpoint еҗҢж—¶иҝӣиЎҢпјҢ Checkpoint barrier N е·Із»ҸжөҒеҲ° Job manager 2 пјҢ Flink job manager еҸҜд»Ҙи§ҰеҸ‘е…¶д»–зҡ„ Checkpoint пјҢжҜ”еҰӮ Checkpoint N + 1 пјҢ Checkpoint N + 2 зӯүзӯүд№ҹеҗҢжӯҘиҝӣиЎҢпјҢеҲ©з”Ёиҝҷз§ҚжңәеҲ¶пјҢеҸҜд»ҘеңЁдёҚйҳ»жҢЎиҝҗз®—зҡ„зҠ¶еҶөдёӢжҢҒз»ӯең°дә§з”ҹ Checkpoint гҖӮ

    3.2 зҠ¶жҖҒз»ҙжҠӨ¶

    зҠ¶жҖҒз»ҙжҠӨеҚіз”ЁдёҖж®өд»Јз ҒеңЁжң¬ең°з»ҙжҠӨзҠ¶жҖҒеҖјпјҢеҪ“зҠ¶жҖҒеҖјйқһеёёеӨ§ж—¶йңҖиҰҒжң¬ең°зҡ„зҠ¶жҖҒеҗҺз«ҜжқҘж”ҜжҢҒгҖӮ

    еҰӮеӣҫпјҢеңЁ Flink зЁӢеәҸдёӯпјҢеҸҜд»ҘйҮҮз”Ё getRuntimeContext().getState(desc); иҝҷз»„ API еҺ»жіЁеҶҢзҠ¶жҖҒгҖӮ Flink жңүеӨҡз§ҚзҠ¶жҖҒеҗҺз«ҜпјҢйҮҮз”Ё API жіЁеҶҢзҠ¶жҖҒеҗҺпјҢиҜ»еҸ–зҠ¶жҖҒж—¶йғҪжҳҜйҖҡиҝҮзҠ¶жҖҒеҗҺз«ҜжқҘиҜ»еҸ–зҡ„гҖӮ Flink жңүдёӨз§ҚдёҚеҗҢзҡ„зҠ¶жҖҒеҖјпјҢд№ҹжңүдёӨз§ҚдёҚеҗҢзҡ„зҠ¶жҖҒеҗҺз«Ҝпјҡ

    • JVM Heap зҠ¶жҖҒеҗҺз«ҜпјҢйҖӮеҗҲж•°йҮҸиҫғе°Ҹзҡ„зҠ¶жҖҒпјҢеҪ“зҠ¶жҖҒйҮҸдёҚеӨ§ж—¶е°ұеҸҜд»ҘйҮҮз”Ё JVM Heap зҡ„зҠ¶жҖҒеҗҺз«ҜгҖӮ JVM Heap зҠ¶жҖҒеҗҺз«ҜдјҡеңЁжҜҸдёҖж¬Ўиҝҗз®—еҖјйңҖиҰҒиҜ»еҸ–зҠ¶жҖҒж—¶пјҢз”Ё Java object read / writes иҝӣиЎҢиҜ»жҲ–еҶҷпјҢдёҚдјҡдә§з”ҹиҫғеӨ§д»Јд»·пјҢдҪҶеҪ“ Checkpoint йңҖиҰҒе°ҶжҜҸдёҖдёӘиҝҗз®—еҖјзҡ„жң¬ең°зҠ¶жҖҒж”ҫе…Ҙ Distributed Snapshots зҡ„ж—¶еҖҷпјҢе°ұйңҖиҰҒиҝӣиЎҢеәҸеҲ—еҢ–дәҶгҖӮ

    • RocksDB зҠ¶жҖҒеҗҺз«ҜпјҢе®ғжҳҜдёҖз§Қ out of core зҡ„зҠ¶жҖҒеҗҺз«ҜгҖӮеңЁ Runtime зҡ„жң¬ең°зҠ¶жҖҒеҗҺз«Ҝи®©дҪҝз”ЁиҖ…еҺ»иҜ»еҸ–зҠ¶жҖҒзҡ„ж—¶еҖҷдјҡз»ҸиҝҮзЈҒзӣҳпјҢзӣёеҪ“дәҺе°ҶзҠ¶жҖҒз»ҙжҠӨеңЁзЈҒзӣҳйҮҢпјҢдёҺд№ӢеҜ№еә”зҡ„д»Јд»·еҸҜиғҪе°ұжҳҜжҜҸж¬ЎиҜ»еҸ–зҠ¶жҖҒж—¶пјҢйғҪйңҖиҰҒз»ҸиҝҮеәҸеҲ—еҢ–е’ҢеҸҚеәҸеҲ—еҢ–зҡ„иҝҮзЁӢгҖӮеҪ“йңҖиҰҒиҝӣиЎҢеҝ«з…§ж—¶еҸӘе°Ҷеә”з”ЁеәҸеҲ—еҢ–еҚіеҸҜпјҢеәҸеҲ—еҢ–еҗҺзҡ„ж•°жҚ®зӣҙжҺҘдј иҫ“еҲ°дёӯеӨ®зҡ„е…ұдә« DFS дёӯгҖӮ

    Flink зӣ®еүҚж”ҜжҢҒд»ҘдёҠдёӨз§ҚзҠ¶жҖҒеҗҺз«ҜпјҢдёҖз§ҚжҳҜзәҜ Memory зҡ„зҠ¶жҖҒеҗҺз«ҜпјҢеҸҰдёҖз§ҚжҳҜжңүиө„жәҗзЈҒзӣҳзҡ„зҠ¶жҖҒеҗҺз«ҜпјҢеңЁз»ҙжҠӨзҠ¶жҖҒж—¶еҸҜд»Ҙж №жҚ®зҠ¶жҖҒзҡ„ж•°йҮҸйҖүжӢ©зӣёеә”зҡ„зҠ¶жҖҒеҗҺз«ҜгҖӮ

    3.3 EventвҖ“Time¶

    3.3.1 дёҚеҗҢж—¶й—ҙз§Қзұ»¶

    еңЁ Flink еҸҠе…¶д»–иҝӣйҳ¶зҡ„жөҒејҸеӨ„зҗҶеј•ж“ҺеҮәзҺ°д№ӢеүҚпјҢеӨ§ж•°жҚ®еӨ„зҗҶеј•ж“ҺдёҖзӣҙеҸӘж”ҜжҢҒ Processing-time зҡ„еӨ„зҗҶгҖӮеҒҮи®ҫе®ҡд№үдёҖдёӘиҝҗз®— Windows зҡ„зӘ—еҸЈпјҢ Windows иҝҗз®—и®ҫе®ҡжҜҸе°Ҹж—¶иҝӣиЎҢз»“з®—гҖӮ Processing-time иҝӣиЎҢиҝҗз®—ж—¶пјҢеҸҜд»ҘеҸ‘зҺ°ж•°жҚ®еј•ж“Һе°Ҷ 3 зӮ№иҮі 4 зӮ№й—ҙж”¶еҲ°зҡ„ж•°жҚ®иҝӣиЎҢз»“з®—гҖӮе®һйҷ…дёҠеңЁеҒҡжҠҘиЎЁжҲ–иҖ…еҲҶжһҗз»“жһңж—¶жҳҜжғідәҶи§Јзңҹе®һдё–з•Ңдёӯ 3 зӮ№иҮі 4 зӮ№д№Ӣй—ҙе®һйҷ…дә§з”ҹж•°жҚ®зҡ„иҫ“еҮәз»“жһңпјҢдәҶи§Је®һйҷ…ж•°жҚ®зҡ„иҫ“еҮәз»“жһңе°ұеҝ…йЎ»йҮҮз”Ё Event-Time дәҶгҖӮ

    еҰӮеӣҫпјҢ Event-Time зӣёеҪ“дәҺдәӢд»¶пјҢе®ғеңЁж•°жҚ®жңҖжәҗеӨҙдә§з”ҹж—¶еёҰжңүж—¶й—ҙжҲіпјҢеҗҺйқўйғҪйңҖиҰҒз”Ёж—¶й—ҙжҲіжқҘиҝӣиЎҢиҝҗз®—гҖӮз”ЁеӣҫжқҘиЎЁзӨәпјҢжңҖејҖе§Ӣзҡ„йҳҹеҲ—ж”¶еҲ°ж•°жҚ®пјҢжҜҸе°Ҹж—¶еҜ№ж•°жҚ®еҲ’еҲҶдёҖдёӘжү№ж¬ЎпјҢиҝҷе°ұжҳҜ Event-Time Process еңЁеҒҡзҡ„дәӢжғ…гҖӮ

    3.3.2 Event-Time еӨ„зҗҶ¶

    Event-Time жҳҜз”ЁдәӢд»¶зңҹе®һдә§з”ҹзҡ„ж—¶й—ҙжҲіеҺ»еҒҡ Re-bucketing пјҢжҠҠеҜ№еә”ж—¶й—ҙ 3 зӮ№еҲ° 4 зӮ№зҡ„ж•°жҚ®ж”ҫеңЁ 3 зӮ№еҲ° 4 зӮ№зҡ„ Bucket пјҢз„¶еҗҺ Bucket дә§з”ҹз»“жһңгҖӮжүҖд»Ҙ Event-Time и·ҹ Processing-time зҡ„жҰӮеҝөжҳҜиҝҷж ·еҜ№жҜ”зҡ„еӯҳеңЁгҖӮ

    Event-Time зҡ„йҮҚиҰҒжҖ§еңЁдәҺи®°еҪ•еј•ж“Һиҫ“еҮәиҝҗз®—з»“жһңзҡ„ж—¶й—ҙгҖӮз®ҖеҚ•жқҘиҜҙпјҢжөҒејҸеј•ж“Һиҝһз»ӯ 24 е°Ҹж—¶еңЁиҝҗиЎҢгҖҒжҗңйӣҶиө„ж–ҷпјҢеҒҮи®ҫ Pipeline йҮҢжңүдёҖдёӘ Windows Operator жӯЈеңЁеҒҡиҝҗз®—пјҢжҜҸе°Ҹж—¶иғҪдә§з”ҹз»“жһңпјҢдҪ•ж—¶иҫ“еҮә Windows зҡ„иҝҗз®—еҖјпјҢиҝҷдёӘж—¶й—ҙзӮ№е°ұжҳҜ Event-Time еӨ„зҗҶзҡ„зІҫй«“пјҢз”ЁжқҘиЎЁзӨәиҜҘж”¶зҡ„ж•°жҚ®е·Із»Ҹж”¶еҲ°гҖӮ

    3.3.3 Watermarks¶

    Flink е®һйҷ…дёҠжҳҜз”Ё Watermarks жқҘе®һзҺ° Event-Time зҡ„еҠҹиғҪгҖӮ Watermarks еңЁ Flink дёӯд№ҹеұһдәҺзү№ж®ҠдәӢд»¶пјҢе…¶зІҫй«“еңЁдәҺеҪ“жҹҗдёӘиҝҗз®—еҖјж”¶еҲ°еёҰжңүж—¶й—ҙжҲі T зҡ„ Watermarks ж—¶е°ұж„Ҹе‘ізқҖе®ғдёҚдјҡжҺҘж”¶еҲ°ж–°зҡ„ж•°жҚ®дәҶгҖӮдҪҝз”Ё Watermarks зҡ„еҘҪеӨ„еңЁдәҺеҸҜд»ҘеҮҶзЎ®йў„дј°ж”¶еҲ°ж•°жҚ®зҡ„жҲӘжӯўж—¶й—ҙгҖӮдёҫдҫӢпјҢеҒҮи®ҫйў„жңҹж”¶еҲ°ж•°жҚ®ж—¶й—ҙдёҺиҫ“еҮәз»“жһңж—¶й—ҙзҡ„ж—¶й—ҙе·®е»¶иҝҹ 5 еҲҶй’ҹпјҢйӮЈд№Ҳ Flink дёӯжүҖжңүзҡ„ Windows Operator жҗңзҙў 3 зӮ№иҮі 4 зӮ№зҡ„ж•°жҚ®пјҢдҪҶеӣ дёәеӯҳеңЁе»¶иҝҹйңҖиҰҒеҶҚеӨҡзӯү 5 еҲҶй’ҹзӣҙиҮіж”¶йӣҶе®Ң 4:05 еҲҶзҡ„ж•°жҚ®пјҢжӯӨж—¶ж–№иғҪеҲӨе®ҡ 4 зӮ№й’ҹзҡ„иө„ж–ҷж”¶йӣҶе®ҢжҲҗдәҶпјҢз„¶еҗҺжүҚдјҡдә§еҮә 3 зӮ№иҮі 4 зӮ№зҡ„ж•°жҚ®з»“жһңгҖӮиҝҷдёӘж—¶й—ҙж®өзҡ„з»“жһңеҜ№еә”зҡ„е°ұжҳҜ Watermarks зҡ„йғЁеҲҶгҖӮ

    3.4 зҠ¶жҖҒдҝқеӯҳдёҺиҝҒз§»¶

    жөҒејҸеӨ„зҗҶеә”з”Ёж— ж—¶ж— еҲ»дёҚеңЁиҝҗиЎҢпјҢиҝҗз»ҙдёҠжңүеҮ дёӘйҮҚиҰҒиҖғйҮҸпјҡ

    • жӣҙж”№еә”з”ЁйҖ»иҫ‘/дҝ®bug зӯүпјҢеҰӮдҪ•е°ҶеүҚдёҖжү§иЎҢзҡ„зҠ¶жҖҒиҝҒз§»еҲ°ж–°зҡ„жү§иЎҢпјҹ

    • еҰӮдҪ•йҮҚж–°е®ҡд№үиҝҗиЎҢзҡ„е№іиЎҢеҢ–зЁӢеәҰпјҹ

    • еҰӮдҪ•еҚҮзә§иҝҗз®—дёӣйӣҶзҡ„зүҲжң¬еҸ·пјҹ

    Checkpoint е®ҢзҫҺз¬ҰеҗҲд»ҘдёҠйңҖжұӮпјҢдёҚиҝҮ Flink дёӯиҝҳжңүеҸҰеӨ–дёҖдёӘеҗҚиҜҚдҝқеӯҳзӮ№пјҲ Savepoint пјүпјҢеҪ“жүӢеҠЁдә§з”ҹдёҖдёӘ Checkpoint зҡ„ж—¶еҖҷпјҢе°ұеҸ«еҒҡдёҖдёӘ Savepoint гҖӮ Savepoint и·ҹ Checkpoint зҡ„е·®еҲ«еңЁдәҺ Checkpoint жҳҜ Flink еҜ№дәҺдёҖдёӘжңүзҠ¶жҖҒеә”з”ЁеңЁиҝҗиЎҢдёӯеҲ©з”ЁеҲҶеёғејҸеҝ«з…§жҢҒз»ӯе‘ЁжңҹжҖ§зҡ„дә§з”ҹ Checkpoint пјҢиҖҢ Savepoint еҲҷжҳҜжүӢеҠЁдә§з”ҹзҡ„ Checkpoint пјҢ Savepoint и®°еҪ•зқҖжөҒејҸеә”з”ЁдёӯжүҖжңүиҝҗз®—е…ғзҡ„зҠ¶жҖҒгҖӮ
