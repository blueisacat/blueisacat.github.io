    Apache Flink йӣ¶еҹәзЎҖе…Ҙй—ЁпјҲ6пјүпјҡFlink Time & Window и§Јжһҗ¶

    1 Window & Time д»Ӣз»Қ¶

    Apache Flink пјҲд»ҘдёӢз®Җз§° Flink пјүжҳҜдёҖдёӘеӨ©з„¶ж”ҜжҢҒж— йҷҗжөҒж•°жҚ®еӨ„зҗҶзҡ„еҲҶеёғејҸи®Ўз®—жЎҶжһ¶пјҢеңЁ Flink дёӯ Window еҸҜд»Ҙе°Ҷж— йҷҗжөҒеҲҮеҲҶжҲҗжңүйҷҗжөҒпјҢжҳҜеӨ„зҗҶжңүйҷҗжөҒзҡ„ж ёеҝғз»„д»¶пјҢзҺ°еңЁ Flink дёӯ Window еҸҜд»ҘжҳҜж—¶й—ҙй©ұеҠЁзҡ„пјҲ Time Window пјүпјҢд№ҹеҸҜд»ҘжҳҜж•°жҚ®й©ұеҠЁзҡ„пјҲ Count Window пјүгҖӮ

    дёӢйқўзҡ„д»Јз ҒжҳҜеңЁ Flink дёӯдҪҝз”Ё Window зҡ„дёӨдёӘзӨәдҫӢ

    2 Window API дҪҝз”Ё¶

    д»Һз¬¬дёҖйғЁеҲҶжҲ‘д»¬е·Із»ҸзҹҘйҒ“ Window зҡ„дёҖдәӣеҹәжң¬жҰӮеҝөпјҢд»ҘеҸҠзӣёе…і API пјҢдёӢйқўжҲ‘д»¬д»ҘдёҖдёӘе®һйҷ…дҫӢеӯҗжқҘзңӢзңӢжҖҺд№ҲдҪҝз”Ё Window зӣёе…ізҡ„ API гҖӮ

    д»Јз ҒжқҘиҮӘ flink-examples

    дёҠйқўзҡ„дҫӢеӯҗдёӯжҲ‘д»¬йҰ–е…ҲдјҡеҜ№жҜҸжқЎж•°жҚ®иҝӣиЎҢж—¶й—ҙжҠҪеҸ–пјҢз„¶еҗҺиҝӣиЎҢ keyby пјҢжҺҘзқҖдҫқж¬Ўи°ғз”Ё window() пјҢ evictor() пјҢ trigger() д»ҘеҸҠ maxBy() гҖӮдёӢйқўжҲ‘д»¬йҮҚзӮ№жқҘзңӢ window() пјҢ evictor() е’Ң trigger() иҝҷеҮ дёӘж–№жі•гҖӮ

    2.1 WindowAssigner, Evictor д»ҘеҸҠ Trigger¶

    2.2 Time & Watermark¶

    дәҶи§Је®ҢдёҠйқўзҡ„еҶ…е®№еҗҺпјҢеҜ№дәҺж—¶й—ҙй©ұеҠЁзҡ„зӘ—еҸЈпјҢжҲ‘д»¬иҝҳжңүдёӨдёӘжҰӮеҝөйңҖиҰҒжҫ„жё…пјҡ Time е’Ң Watermark гҖӮ

    жҲ‘д»¬зҹҘйҒ“еңЁеҲҶеёғејҸзҺҜеўғдёӯ Time жҳҜдёҖдёӘеҫҲйҮҚиҰҒзҡ„жҰӮеҝөпјҢеңЁ Flink дёӯ Time еҸҜд»ҘеҲҶдёәдёүз§Қ Event-Time пјҢ Processing-Time д»ҘеҸҠ Ingestion-Time пјҢдёүиҖ…зҡ„е…ізі»жҲ‘д»¬еҸҜд»Ҙд»ҺдёӢеӣҫдёӯеҫ—зҹҘпјҡ

    Event Time гҖҒ Ingestion Time гҖҒ Processing Time

    Event-Time иЎЁзӨәдәӢд»¶еҸ‘з”ҹзҡ„ж—¶й—ҙпјҢ Processing-Time еҲҷиЎЁзӨәеӨ„зҗҶж¶ҲжҒҜзҡ„ж—¶й—ҙпјҲеўҷдёҠж—¶й—ҙпјүпјҢ Ingestion-Time иЎЁзӨәиҝӣе…ҘеҲ°зі»з»ҹзҡ„ж—¶й—ҙгҖӮ

    еңЁ Flink дёӯжҲ‘д»¬еҸҜд»ҘйҖҡиҝҮдёӢйқўзҡ„ж–№ејҸиҝӣиЎҢ Time зұ»еһӢзҡ„и®ҫзҪ® env.setStreamTimeCharacteristic(TimeCharacteristic.ProcessingTime); // и®ҫзҪ®дҪҝз”Ё ProcessingTime

    дәҶи§ЈдәҶ Time д№ӢеҗҺпјҢжҲ‘д»¬иҝҳйңҖиҰҒзҹҘйҒ“ Watermark зӣёе…ізҡ„жҰӮеҝөгҖӮ

    жҲ‘д»¬еҸҜд»ҘиҖғиҷ‘дёҖдёӘиҝҷж ·зҡ„дҫӢеӯҗпјҡжҹҗ App дјҡи®°еҪ•з”ЁжҲ·зҡ„жүҖжңүзӮ№еҮ»иЎҢдёәпјҢе№¶еӣһдј ж—Ҙеҝ—пјҲеңЁзҪ‘з»ңдёҚеҘҪзҡ„жғ…еҶөдёӢпјҢе…ҲдҝқеӯҳеңЁжң¬ең°пјҢе»¶еҗҺеӣһдј пјүгҖӮ A з”ЁжҲ·еңЁ 11:02 еҜ№ App иҝӣиЎҢж“ҚдҪңпјҢ B з”ЁжҲ·еңЁ 11:03 ж“ҚдҪңдәҶ App пјҢдҪҶжҳҜ A з”ЁжҲ·зҡ„зҪ‘з»ңдёҚеӨӘзЁіе®ҡпјҢеӣһдј ж—Ҙеҝ—е»¶иҝҹдәҶпјҢеҜјиҮҙжҲ‘д»¬еңЁжңҚеҠЎз«Ҝе…ҲжҺҘеҸ—еҲ° B з”ЁжҲ· 11:03 зҡ„ж¶ҲжҒҜпјҢз„¶еҗҺеҶҚжҺҘеҸ—еҲ° A з”ЁжҲ· 11:02 зҡ„ж¶ҲжҒҜпјҢж¶ҲжҒҜд№ұеәҸдәҶгҖӮ

    йӮЈжҲ‘д»¬жҖҺд№ҲдҝқиҜҒеҹәдәҺ event-time зҡ„зӘ—еҸЈеңЁй”ҖжҜҒзҡ„ж—¶еҖҷпјҢе·Із»ҸеӨ„зҗҶе®ҢдәҶжүҖжңүзҡ„ж•°жҚ®е‘ўпјҹиҝҷе°ұжҳҜ watermark зҡ„еҠҹиғҪжүҖеңЁгҖӮ watermark дјҡжҗәеёҰдёҖдёӘеҚ•и°ғйҖ’еўһзҡ„ж—¶й—ҙжҲі t пјҢ watermark(t) иЎЁзӨәжүҖжңүж—¶й—ҙжҲідёҚеӨ§дәҺ t зҡ„ж•°жҚ®йғҪе·Із»ҸеҲ°жқҘдәҶпјҢжңӘжқҘе°ҸдәҺзӯүдәҺ t зҡ„ж•°жҚ®дёҚдјҡеҶҚжқҘпјҢеӣ жӯӨеҸҜд»Ҙж”ҫеҝғең°и§ҰеҸ‘е’Ңй”ҖжҜҒзӘ—еҸЈдәҶгҖӮдёӢеӣҫдёӯз»ҷдәҶдёҖдёӘд№ұеәҸж•°жҚ®жөҒдёӯзҡ„ watermark дҫӢеӯҗ

    2.3 иҝҹеҲ°зҡ„ж•°жҚ®¶

    дёҠйқўзҡ„ watermark и®©жҲ‘д»¬иғҪеӨҹеә”еҜ№д№ұеәҸзҡ„ж•°жҚ®пјҢдҪҶжҳҜзңҹе®һдё–з•ҢдёӯжҲ‘д»¬жІЎжі•еҫ—еҲ°дёҖдёӘе®ҢзҫҺзҡ„ watermark ж•°еҖјпјҢиҰҒд№ҲжІЎжі•иҺ·еҸ–еҲ°пјҢиҰҒд№ҲиҖ—иҙ№еӨӘеӨ§пјҢеӣ жӯӨе®һйҷ…е·ҘдҪңдёӯжҲ‘д»¬дјҡдҪҝз”Ёиҝ‘дјј watermark - з”ҹжҲҗ watermark(t) д№ӢеҗҺпјҢиҝҳжңүиҫғе°Ҹзҡ„жҰӮзҺҮжҺҘеҸ—еҲ°ж—¶й—ҙжҲі t д№ӢеүҚзҡ„ж•°жҚ®пјҢеңЁ Flink дёӯе°Ҷиҝҷдәӣж•°жҚ®е®ҡд№үдёә late elements пјҢеҗҢж ·жҲ‘д»¬еҸҜд»ҘеңЁ window дёӯжҢҮе®ҡжҳҜе…Ғи®ёе»¶иҝҹзҡ„жңҖеӨ§ж—¶й—ҙпјҲй»ҳи®Өдёә 0 пјүпјҢеҸҜд»ҘдҪҝз”ЁдёӢйқўзҡ„д»Јз ҒиҝӣиЎҢи®ҫзҪ®

    и®ҫзҪ®В allowedLatenessВ д№ӢеҗҺпјҢиҝҹжқҘзҡ„ж•°жҚ®еҗҢж ·еҸҜд»Ҙи§ҰеҸ‘зӘ—еҸЈпјҢиҝӣиЎҢиҫ“еҮәпјҢеҲ©з”Ё Flink зҡ„ side output жңәеҲ¶пјҢжҲ‘д»¬еҸҜд»ҘиҺ·еҸ–еҲ°иҝҷдәӣиҝҹеҲ°зҡ„ж•°жҚ®пјҢдҪҝз”Ёж–№ејҸеҰӮдёӢпјҡ

    йңҖиҰҒжіЁж„Ҹзҡ„жҳҜпјҢи®ҫзҪ®дәҶ allowedLateness д№ӢеҗҺпјҢиҝҹеҲ°зҡ„ж•°жҚ®д№ҹеҸҜиғҪи§ҰеҸ‘зӘ—еҸЈпјҢеҜ№дәҺ Session window жқҘиҜҙпјҢеҸҜиғҪдјҡеҜ№зӘ—еҸЈиҝӣиЎҢеҗҲе№¶пјҢдә§з”ҹйў„жңҹеӨ–зҡ„иЎҢдёәгҖӮ

    3 Window еҶ…йғЁе®һзҺ°¶

    еңЁи®Ёи®ә Window еҶ…йғЁе®һзҺ°зҡ„ж—¶еҖҷпјҢжҲ‘д»¬еҶҚйҖҡиҝҮдёӢеӣҫеӣһйЎҫдёҖдёӢ Window зҡ„з”ҹе‘Ҫе‘Ёжңҹ

    жҜҸжқЎж•°жҚ®иҝҮжқҘд№ӢеҗҺпјҢдјҡз”ұ WindowAssigner еҲҶй…ҚеҲ°еҜ№еә”зҡ„ Window пјҢеҪ“ Window иў«и§ҰеҸ‘д№ӢеҗҺпјҢдјҡдәӨз»ҷ Evictor пјҲеҰӮжһңжІЎжңүи®ҫзҪ® Evictor еҲҷи·іиҝҮпјүпјҢз„¶еҗҺеӨ„зҗҶ UserFunction гҖӮе…¶дёӯ WindowAssigner пјҢ Trigger пјҢ Evictor жҲ‘д»¬йғҪеңЁдёҠйқўи®Ёи®әиҝҮпјҢиҖҢ UserFunction еҲҷжҳҜз”ЁжҲ·зј–еҶҷзҡ„д»Јз ҒгҖӮ

    ж•ҙдёӘжөҒзЁӢиҝҳжңүдёҖдёӘй—®йўҳйңҖиҰҒи®Ёи®әпјҡ Window дёӯзҡ„зҠ¶жҖҒеӯҳеӮЁгҖӮжҲ‘д»¬зҹҘйҒ“ Flink жҳҜж”ҜжҢҒ Exactly Once еӨ„зҗҶиҜӯд№үзҡ„пјҢйӮЈд№Ҳ Window дёӯзҡ„зҠ¶жҖҒеӯҳеӮЁе’Ңжҷ®йҖҡзҡ„зҠ¶жҖҒеӯҳеӮЁеҸҲжңүд»Җд№ҲдёҚдёҖж ·зҡ„ең°ж–№е‘ўпјҹ

    йҰ–е…Ҳз»ҷеҮәе…·дҪ“зҡ„зӯ”жЎҲпјҡд»ҺжҺҘеҸЈдёҠеҸҜд»Ҙи®ӨдёәжІЎжңүеҢәеҲ«пјҢдҪҶжҳҜжҜҸдёӘ Window дјҡеұһдәҺдёҚеҗҢзҡ„ namespace пјҢиҖҢйқһ Window еңәжҷҜдёӢпјҢеҲҷйғҪеұһдәҺ VoidNamespace пјҢжңҖз»Ҳз”ұ State/Checkpoint жқҘдҝқиҜҒж•°жҚ®зҡ„ Exactly Once иҜӯд№үпјҢдёӢйқўжҲ‘д»¬д»Һ org.apache.flink.streaming.runtime.operators.windowing.WindowOperator ж‘ҳеҸ–дёҖж®өд»Јз ҒиҝӣиЎҢйҳҗиҝ°

    д»ҺдёҠйқўжҲ‘д»¬еҸҜд»ҘзҹҘйҒ“пјҢ Window дёӯзҡ„зҡ„е…ғзҙ еҗҢж ·жҳҜйҖҡиҝҮ state иҝӣиЎҢз»ҙжҠӨпјҢз„¶еҗҺз”ұ Checkpoint жңәеҲ¶дҝқиҜҒ Exactly Once иҜӯд№үгҖӮ

    Time гҖҒ Window зӣёе…ізҡ„жүҖжңүеҶ…е®№йғҪе·Із»Ҹи®Іи§Је®ҢжҜ•пјҢдё»иҰҒеҢ…жӢ¬дёәд»Җд№ҲиҰҒжңү Window пјӣ Window дёӯзҡ„дёүдёӘж ёеҝғз»„д»¶пјҡ WindowAssigner гҖҒ Trigger е’Ң Evictor пјӣ Window дёӯжҖҺд№ҲеӨ„зҗҶд№ұеәҸж•°жҚ®пјҢд№ұеәҸж•°жҚ®жҳҜеҗҰе…Ғи®ёе»¶иҝҹпјҢд»ҘеҸҠжҖҺд№ҲеӨ„зҗҶиҝҹеҲ°зҡ„ж•°жҚ®пјӣжңҖеҗҺжҲ‘д»¬жўізҗҶдәҶж•ҙдёӘ Window зҡ„ж•°жҚ®жөҒзЁӢпјҢд»ҘеҸҠ Window дёӯжҖҺд№ҲдҝқиҜҒ Exactly Once иҜӯд№үгҖӮ
